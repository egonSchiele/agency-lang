# Classes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add OOP classes to the Agency language with fields, constructors, methods, single inheritance, and serialization support.

**Architecture:** Classes are a new AST node type (`ClassDefinition`) that compiles to TypeScript classes via the builder. The runtime gets a class registry on `RuntimeContext` for serialization/deserialization. The type checker, symbol table, and program info collector are extended to handle class definitions.

**Tech Stack:** tarsec (parsing), vitest (testing), TypeScript IR (code generation)

**Spec:** `docs/superpowers/specs/2026-04-05-classes-design.md`

- [ ] **Step 3: Implement the parser**

Create `lib/parsers/classDefinition.ts`. The parser must handle:

1. `class` keyword
2. Class name (identifier)
3. Optional `extends ParentName`
4. Opening `{`
5. Fields: `name: type` (one per line)
6. Optional `constructor(params) { body }`
7. Methods: `name(params): returnType { body }` (zero or more)
8. Closing `}`

Use tarsec combinators: `seqC`, `capture`, `str`, `or`, `many`, `optional`, `map`, `regex`. Refer to `lib/parsers/function.ts` for how to parse function parameters and bodies (reuse `functionParameterParser` and the body parsing approach). Refer to `lib/parsers/typeHints.ts` for `variableTypeParser`.

Key details:
- Fields are `identifier : variableType` followed by newline
- Constructor uses the same parameter parser as functions
- Constructor body needs to handle `this.field = value` and `super(...)` — these are already parseable as assignments and function calls via the existing `nodeParser`
- Methods are `identifier ( params ) : returnType { body }` — similar to function definitions but without the `def` keyword
- The body of constructor/methods is parsed using the same `agencyNode` many-parser used elsewhere

- [ ] **Step 3: Implement createClassReviver**

```typescript
export type ClassRegistry = Record<string, { fromJSON: (data: any) => any }>;

export function createClassReviver(classRegistry: ClassRegistry) {
  return function reviver(key: string, value: any): any {
    if (value && typeof value === "object" && "__class" in value) {
      const cls = classRegistry[value.__class];
      if (cls) return cls.fromJSON(value);
    }
    return value;
  };
}
```
