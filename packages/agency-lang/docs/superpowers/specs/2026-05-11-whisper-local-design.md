# Local Whisper package design

> **Status: shipped — historical design document.** This spec records the
> design as initially proposed. The implementation has since diverged in
> several places. **Authoritative current behavior lives in
> [`packages/whisper-local/README.md`](../../../whisper-local/README.md)
> and [`packages/whisper-local/docs/DEV.md`](../../../whisper-local/docs/DEV.md).**
>
> Notable differences from this spec as shipped:
> - **Default model is `base.en`** (this spec says `base`). All ten upstream
>   whisper.cpp models (`tiny` through `large-v3-turbo`) are in
>   `KNOWN_MODELS` and pinned in `models.lock.json` with verified SHAs.
> - **No `install` / `postinstall` script.** Users explicitly run
>   `npx agency-whisper build` after install. (This spec's "Build and
>   install" section pre-dates that decision.)
> - **No final-file `chmod 0644`.** The downloader writes via
>   `createWriteStream` + `rename`; permissions are whatever the user's
>   umask produces.
> - **No `tests/agency-js/transcribe.test.ts`.** That harness style does
>   not exist in agency-lang; see `tests/agency-js/README.md` in the
>   package for the rationale.
> - **No `index.js`.** The `package.json` points `main` directly at
>   `dist/src/transcribe.js`.
> - **Additional ffmpeg hardening** (protocol whitelist, timeout, output
>   cap, path validation) and a **bounded LRU handle cache** were added
>   during code review; see DEV.md §7 and §7a.

## Motivation

Agency's stdlib `transcribe(filepath, language)` sends audio to OpenAI's Whisper API. That's the right default for most users, but it has three drawbacks for some:

- **Cost** — every transcription costs money.
- **Privacy** — audio leaves the user's machine.
- **Network** — no offline transcription, and latency floors on the round trip.

Whisper is also a publicly-released model that runs locally via [whisper.cpp](https://github.com/ggml-org/whisper.cpp). This spec adds `@agency-lang/whisper-local`, a new Agency package that exposes `transcribe()` running entirely on the user's machine.

Existing npm wrappers (`nodejs-whisper`, `smart-whisper`, `@lumen-labs-dev/whisper-node`) are small, sparsely-maintained projects from unknown authors. Rather than trust them, we write our own thin N-API bindings to whisper.cpp following the architecture used by [node-llama-cpp](https://github.com/withcatai/node-llama-cpp).

## Design principles

1. **Mirror the stdlib API.** `transcribe(filepath, language, model)` differs from the stdlib only in the added `model` parameter. Switching between cloud and local is a one-line import change.
2. **No interrupt gate.** The stdlib version interrupts because audio leaves the machine. Local Whisper doesn't, so we drop the gate to match other local operations like `read()` and `print()`.
3. **Vendored, deterministic builds.** Whisper.cpp source lives in this repo. Install requires no network. No git submodules.
4. **Integrity-first model downloads.** Every model has a committed SHA-256. Downloads from a commit-pinned HuggingFace URL. Failure to match the expected hash is fatal.
5. **Source-build only, v0.** Users need cmake + a C++17 compiler. Prebuilt binaries are a follow-up using the optional-deps matrix pattern.

## Component layering

Three layers; each understandable and testable in isolation.

**1. Addon (`src/addon.cc`)** — Thin N-API wrapper exposing exactly three functions:

```cpp
loadModel(path: string)            → ModelHandle      // wraps whisper_init_from_file_with_params
transcribe(handle, pcm: Float32Array, opts)
                                   → string[]         // wraps whisper_full + segment iteration
freeModel(handle)                  → void             // wraps whisper_free
```

Synchronous whisper.cpp calls run on a libuv worker thread via `Napi::AsyncWorker` so the event loop stays responsive. The addon has no knowledge of model paths, downloads, or audio formats — it operates on already-decoded PCM and an already-loaded model.

**2. TS layer (`src/transcribe.ts`, `modelManager.ts`, `ffmpeg.ts`)** — Pure JavaScript orchestration. No native code dependency in `modelManager.ts` or `ffmpeg.ts`, so they're trivially unit-testable.

**3. Agency wrapper (`index.agency`)** — Single exported function matching the stdlib signature plus a `model` parameter.

## Public API

```ts
// In Agency code:
import { transcribe } from "pkg::@agency-lang/whisper-local"

node main() {
  const text = transcribe("interview.m4a", "en", "base.en")
  print(text)
}
```

```ts
// index.agency
export def transcribe(filepath: string, language: string = "", model: string = "base"): string {
  """
  Transcribe an audio file to text using a local Whisper model (whisper.cpp).
  No network, no API key. First call downloads the model if not present.

  @param filepath - Path to the audio file (any format ffmpeg supports)
  @param language - Language code for accuracy (e.g. "en"); empty = auto-detect
  @param model    - Whisper model name. Defaults to "base" (multilingual, 142 MB).
                    Choices: tiny, tiny.en, base, base.en, small, small.en,
                    medium, medium.en, large-v3, large-v3-turbo
  """
  return _transcribe(filepath, language, model)
}
```

No interrupt. No `apiKey`. Default model is `base` (multilingual) — slightly less accurate on English than `base.en`, but works for any language input without the user reasoning about the default.

## Data flow

```
.agency transcribe(file, lang, model)
        │
        ▼  (transcribe.ts)
  resolveModelPath(model)
        │   └─ if missing → download(modelsLock[model]) → verify SHA-256 → atomic rename
        ▼
  decodeToPcm(file)
        │   └─ spawn `ffmpeg -i file -ac 1 -ar 16000 -f f32le -loglevel error -`
        │      → read stdout to end → wrap in Float32Array
        ▼
  loadModel(modelPath)            ── process-cached: same path = reuse handle
        │
        ▼
  addon.transcribe(handle, pcm, { language })
        │
        ▼
  segments.join("") → return
```

Model handles are cached process-wide keyed by absolute model path. Repeated `transcribe()` calls in the same Agency run don't re-load the model. The cache survives interrupts because handles aren't part of Agency checkpoint state — they're rebuilt lazily on next call if the process is replaced.

## Build and install

- `cmake-js` is a runtime dep. Native build runs as part of the package's `install` script: `cmake-js compile --runtime=node`.
- `CMakeLists.txt` adds `vendor/whisper.cpp/src/` and `vendor/whisper.cpp/ggml/src/` as static libraries, then links the addon (`src/addon.cc`).
- macOS: Metal enabled via `WHISPER_METAL=ON`. CMake auto-detects the Metal framework.
- Linux/Windows: CPU only for v0.
- Output: `build/Release/whisper_addon.node`, loaded by `src/transcribe.ts` via `require`.
- `make submodules` is not needed (no submodules). `make` in the package directory: builds .ts → .js, builds the .agency wrapper, runs cmake-js.
- Install fails with a clear, per-platform error if `cmake` or a C++17 compiler isn't found. README lists install commands for macOS (`xcode-select --install && brew install cmake`), Debian (`apt install cmake build-essential`), Fedora (`dnf install cmake gcc-c++`).

## Vendoring whisper.cpp

The source lives at `vendor/whisper.cpp/` with a `VERSION` file recording the upstream tag (initially `v1.7.6` or current at implementation time). Vendoring is manual:

1. Download upstream tarball for the pinned tag.
2. Verify upstream tag's SHA-256 against a value we record in `vendor/whisper.cpp/UPSTREAM_SHA256`.
3. Extract `src/`, `include/`, `ggml/src/`, `ggml/include/` into `vendor/whisper.cpp/`.
4. Commit. PR description includes the upstream changelog summary.

Bumps are intentional, reviewed, and rare. This is fine because whisper.cpp itself moves slowly and we don't track unstable APIs (we only call the six C functions identified in research).

## Model management

**Location:** `~/.agency/models/whisper/<name>.bin`. Override via `AGENCY_WHISPER_MODELS_DIR`. The `~/.agency/` root is a new convention this package establishes for Agency-related caches; future packages can store siblings (e.g. `~/.agency/models/embeddings/`).

**Source of truth (`models.lock.json`):**

```json
{
  "schemaVersion": 1,
  "models": {
    "tiny":         { "url": "https://huggingface.co/ggerganov/whisper.cpp/resolve/<commit>/ggml-tiny.bin",         "sha256": "...", "sizeBytes": 77691000 },
    "tiny.en":      { "url": "...", "sha256": "...", "sizeBytes": 77691000 },
    "base":         { "url": "...", "sha256": "...", "sizeBytes": 147951000 },
    "base.en":      { "url": "...", "sha256": "...", "sizeBytes": 147951000 },
    "small":        { "url": "...", "sha256": "...", "sizeBytes": 487601000 },
    "small.en":     { "url": "...", "sha256": "...", "sizeBytes": 487601000 },
    "medium":       { "url": "...", "sha256": "...", "sizeBytes": 1533000000 },
    "medium.en":    { "url": "...", "sha256": "...", "sizeBytes": 1533000000 },
    "large-v3":     { "url": "...", "sha256": "...", "sizeBytes": 3094000000 },
    "large-v3-turbo": { "url": "...", "sha256": "...", "sizeBytes": 1620000000 }
  }
}
```

URLs use a commit SHA, not `resolve/main`, so the URL refers to an immutable snapshot of the HuggingFace repo state.

**Download flow:**

1. Stream to `~/.agency/models/whisper/<name>.bin.partial`. Compute SHA-256 incrementally.
2. On completion, compare digest to the lockfile entry. Mismatch → delete partial → throw with a security-relevant error message.
3. On match, atomic rename to `<name>.bin`. Set file mode to `0644`.
4. Progress is reported to stderr if `process.stderr.isTTY`.

**Manual placement** (README documents this clearly): user runs `mkdir -p ~/.agency/models/whisper/` and drops in `ggml-<name>.bin` themselves. On-disk files are **trusted by default** — we don't hash them on every cold start (hashing a 3 GB `large-v3` would add seconds to the first call). Verification is explicit: the user runs `agency-whisper verify <name>` to confirm the file matches the lockfile hash. Files that we downloaded ourselves were always verified at download time, so this only matters for manually-placed files.

This split — auto-verify what we control, trust what the user controls, expose a `verify` command for paranoid use — keeps the cold-start fast while still letting users audit on demand.

**CLI:** `npx -p @agency-lang/whisper-local agency-whisper pull <name>` — pre-downloads a model without running transcription. Also `agency-whisper list` (shows installed models + sizes) and `agency-whisper verify <name>` (re-hashes an installed file).

## Audio decoding

We shell out to `ffmpeg`:

```
ffmpeg -hide_banner -loglevel error \
       -i <filepath> \
       -ac 1 -ar 16000 -f f32le -
```

This reads any format ffmpeg supports, downmixes to mono, resamples to 16 kHz, emits raw little-endian float32 PCM on stdout. We read stdout to end into a Buffer and wrap as `Float32Array`. Stderr is captured for error messages.

**Why ffmpeg, not SoX or a JS decoder:** ffmpeg's codec coverage is universal (mp3, m4a, mp4, ogg, flac, webm, wav, …). SoX needs `libsox-fmt-all` for m4a/aac and is less common on dev machines. A pure-JS decoder would bundle 5+ codec libraries.

**Why shelling out doesn't infect our license:** ffmpeg is dual-licensed LGPL/GPL, and the binary most users install is the GPL build. Copyleft propagates through linking, not through spawning a separately-installed process. Same setup as the stdlib's existing SoX dependency for `record()`.

Missing ffmpeg → we throw with a per-platform install hint (`brew install ffmpeg`, `apt install ffmpeg`, etc.) detected from `process.platform`.

## Error handling

Errors propagate as thrown `Error` objects from the JS layer. Per Agency's convention, a function that throws produces a `failure` value at the call site. Specific cases:

- Missing ffmpeg: `Error: ffmpeg not found on PATH. Install with 'brew install ffmpeg' (macOS) or 'apt install ffmpeg' (Debian/Ubuntu).`
- Unknown model name: `Error: unknown model "<name>". Choices: tiny, tiny.en, base, base.en, …`
- Network failure mid-download: partial file deleted, original error chained: `Error: failed to download model "base": <cause>`
- SHA-256 mismatch: `Error: SHA-256 mismatch for model "base" (expected <a>, got <b>). The downloaded file has been deleted. This may indicate a corrupted download or a compromised mirror.`
- whisper.cpp error: addon converts the C return code into an `Error` with a human-readable message.
- Invalid audio file: ffmpeg's stderr is included in the thrown error.

## Testing

- **Unit (vitest, fast, always run):**
  - `modelManager.test.ts` — path resolution, lockfile parsing, URL construction, SHA-256 verification against fixture bytes.
  - `ffmpeg.test.ts` — arg construction, error messages on missing binary (mocked `spawn`).
  - `transcribe.test.ts` — full TS-layer flow with the addon mocked, asserting the right args reach each layer.
- **Integration (vitest, gated by `AGENCY_RUN_SLOW=1`):**
  - Checks in a 5-second sample WAV (`tests/fixtures/hello.wav`, ~150 KB, public-domain content).
  - Downloads `tiny.en` (~75 MB) on first run, caches in `~/.agency/models/whisper/`.
  - Asserts `transcribe("hello.wav", "en", "tiny.en")` contains the expected words.
- **Agency-js test (`tests/agency-js/transcribe.test.ts`):** Exercises the `.agency` wrapper end-to-end by importing a stub JS module that records the args the wrapper passed through. Confirms `language` and `model` defaults reach the JS layer correctly. No inference, no native addon load.

CI runs the unit + Agency tests on every PR. The integration test runs nightly and on release branches.

## Out of scope for v0

- **Prebuilt binaries.** Will land as a follow-up via the optional-deps matrix pattern (sibling packages `@agency-lang/whisper-local-darwin-arm64`, `-linux-x64`, etc.). The main package's loader already needs to handle "addon not pre-installed → source build" — we just add a "try sibling packages first" probe.
- **Streaming / partial results.** whisper.cpp supports it; not needed for v0's transcription use case.
- **Speaker diarization.** Out-of-scope feature.
- **Windows source-build support.** Documented as Linux/macOS only for v0. Windows will arrive with the prebuilt matrix.
- **Translation mode.** Trivially adds via `whisper_full_params.translate = true`, but not part of v0's surface.
- **CUDA / Vulkan.** macOS Metal is free. CUDA/Vulkan are additional CMake flags; deferred to prebuilts.

## Open questions

None blocking. A few minor decisions resolved in the design above (default model = `base`, package name = `whisper-local`, cache root = `~/.agency/`) but worth re-litigating if assumptions change.
